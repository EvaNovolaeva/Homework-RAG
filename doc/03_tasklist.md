## Итерация 0 - Planning

### Задачи
- Создать `doc/00_project_idea.md`
- Создать `doc/01_vision.md`
- Создать `doc/02_conventions.md`
- Создать `doc/03_tasklist.md`

### Проверка
- Все planning-файлы существуют в репозитории
- В каждом файле заполнены обязательные разделы

## Итерация 1 - Data loading

### Задачи
- Скачать или подключить выбранный датасет
- Написать код для чтения данных
- Проверить названия колонок и формат записей
- Подготовить базовую функцию загрузки данных

### Проверка
- Скрипт загружает датасет без ошибок
- В консоли или notebook видно количество строк и список колонок

## Итерация 2 - Text transform

### Задачи
- Преобразовать каждую строку датасета в текстовое описание
- Определить, какие поля включать в текст
- Очистить пустые или некорректные значения
- Подготовить итоговый список текстовых документов для индексации

### Проверка
- Для нескольких строк можно вывести готовые текстовые описания
- Текст читаемый и отражает ключевые признаки записи

## Итерация 3 - Indexing

### Задачи
- Выбрать модель эмбеддингов
- Построить эмбеддинги для текстовых описаний
- Создать векторный индекс
- Сохранить индекс локально для повторного использования

### Проверка
- Индекс создаётся без ошибок
- После повторного запуска индекс можно загрузить из файла
- Количество элементов в индексе совпадает с количеством подготовленных текстов

## Итерация 4 - Retrieval

### Задачи
- Реализовать поиск top-k релевантных фрагментов
- Настроить преобразование вопроса пользователя в embedding
- Вернуть найденные документы вместе с метаданными
- Проверить качество retrieval на простых запросах

### Проверка
- По тестовому вопросу система возвращает несколько релевантных текстов
- Найденные фрагменты логически связаны с вопросом

## Итерация 5 - Generation

### Задачи
- Подключить LLM для генерации ответа
- Собрать prompt с вопросом и найденным контекстом
- Реализовать правило ответа только по контексту
- Добавить отказ, если релевантные данные не найдены

### Проверка
- На вопрос с релевантным контекстом система даёт осмысленный ответ
- На вопрос вне данных система отказывается отвечать корректно

## Итерация 6 - UI

### Задачи
- Сделать простой интерфейс для ввода вопроса
- Отображать найденный контекст и итоговый ответ
- Подготовить минимально удобный сценарий запуска для пользователя

### Проверка
- Пользователь может запустить интерфейс локально
- Через интерфейс можно ввести вопрос и получить ответ

## Итерация 7 - Testing

### Задачи
- Подготовить набор тестовых вопросов
- Проверить сценарий с полным ответом
- Проверить сценарий с частичным ответом
- Проверить сценарий с отказом при отсутствии данных
- Проверить стабильность запуска проекта

### Проверка
- Все тестовые сценарии выполняются
- Поведение системы соответствует правилам из `conventions.md`

## Итерация 8 - MVP polish

### Задачи
- Исправить ошибки, найденные на тестировании
- Упростить запуск проекта
- Обновить README и инструкции
- Проверить, что весь MVP работает end-to-end

### Проверка
- Новый пользователь может запустить проект по инструкции
- MVP проходит демонстрационный сценарий от загрузки данных до ответа на вопрос

## Критерии завершения MVP

MVP считается завершённым, если:
- проект загружает выбранный датасет;
- строки датасета преобразуются в текстовый вид;
- текстовые данные индексируются;
- пользователь может задать вопрос;
- система выполняет retrieval по индексу;
- LLM формирует ответ на основе найденного контекста;
- при отсутствии данных система делает корректный отказ;
- проект можно локально запустить по инструкции без сложной настройки.

## Как проверить

Файл `tasklist.md` считается корректным, если:
- в нём есть итерации 0–8;
- для каждой итерации указаны задачи и понятная команда проверки;
- явно указана текущая итерация;
- описаны критерии завершения MVP;
- по этому плану можно двигаться по проекту последовательно, без пропуска шагов.
